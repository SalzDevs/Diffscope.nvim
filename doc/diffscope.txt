INTRODUCTION                                        *diffscope-introduction*

Diffscope.nvim opens changed files in a dedicated left viewer / right editor
workspace. The left pane is a read-only generated code view with diff colors.
The right pane is the regular editable file buffer.

The left pane is intentionally not raw unified diff text. It has no @@ headers
and no +/- body prefixes. It looks like code: removed lines are inserted in
place and highlighted red, added lines are highlighted green, and unchanged
lines are shown normally.

Diffscope intentionally exposes |:DiffScope| only. It does not install a
`:Diff` alias.

==============================================================================
COMMANDS                                                *diffscope-commands*

:DiffScope                                              *:DiffScope*
        Open a dedicated Diffscope tab for the changed files in the repository.
        If the current file has changes, it is selected first; otherwise
        Diffscope opens the first changed file.

:DiffScope staged
        Show staged changes in the left read-only code view while keeping the
        worktree file editable on the right.

:DiffScope %
        Open a diff code view for the current file.

:DiffScope {left} {right}
        Compare two files directly. The left pane is the read-only diff code
        view; the right file is editable.

==============================================================================
MAPPINGS                                                *diffscope-mappings*

Default normal-mode mappings inside Diffscope buffers:

                                                        *diffscope-mappings-q*
q       Close the diff viewer.

                                                        *diffscope-mappings-help*
?       Toggle help.

                                                        *diffscope-mappings-files*
f       Open changed-files picker.
]f      Open next changed file.
[f      Open previous changed file.
/       Filter the changed-files picker.
d       Toggle reviewed marker for the current/selected file.

                                                        *diffscope-mappings-hunks*
]c      Jump to the next hunk.
[c      Jump to the previous hunk.

                                                        *diffscope-mappings-reload*
R       Reload external changes detected while Diffscope is open. Also works
        from the changed-files picker.

                                                        *diffscope-mappings-stage*
s       Write and stage the current file.

                                                        *diffscope-mappings-reset*
r       Reset the current file after confirmation.

==============================================================================
CONFIGURATION                                      *diffscope-configuration*

Example: >lua

  require("diffscope").setup({
    layout = {
      base_width = nil, -- nil keeps viewer/editor panes equal
    },
    mappings = {
      close = "q",
      help = "?",
      next_hunk = "]c",
      prev_hunk = "[c",
      files = "f",
      next_file = "]f",
      prev_file = "[f",
      reload = "R",
      toggle_reviewed = "d",
      picker_filter = "/",
      stage_file = "s",
      reset_file = "r",
    },
  })
<

==============================================================================
DESIGN NOTES                                            *diffscope-design*

Diffscope does not open a separate review list or a raw diff buffer. The left
pane should feel like a normal read-only code buffer with diff colors. The
right pane is the user's real file buffer.

The left pane refreshes after the editable buffer is written. File navigation
is built into Diffscope through a changed-files picker and next/previous file
mappings, so the workflow does not depend on external file explorers. The
picker shows review progress, additions/deletions, stale markers, reviewed
markers, and an interactive filter.

When an agent or another process changes files while Diffscope is open,
Diffscope marks both panes as stale and prompts the user to press R. The
changed-files picker marks externally updated files with ●. If the editable
buffer has no unsaved edits, Diffscope also auto-refreshes the left code view.
Reloading refreshes the changed-file list and the current diff without silently
overwriting unsaved user edits.
